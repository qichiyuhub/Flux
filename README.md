
# **Flux**

一个轻量、高性能的 Cloudflare Workers 反向代理脚本。它通过创新的**路径静默鉴权**机制，在保障安全不滥用的前提下，提供无参数污染的访问透传服务。

## 核心特性

- **访问突破**：解除网络策略与地域限制，实现对受阻资源的无障碍访问。
- **隐私护航**：自动剥离 `X-Forwarded-For` 等特征头部，隐匿真实 IP，保障浏览隐私。
- **全球加速**：依托 Cloudflare 庞大的边缘网络，显著降低延迟，提升资源加载速度。
- **CORS 全解**：完美支持跨域资源共享，自动处理预检请求，满足复杂的前端调用需求。
- **静默鉴权**：采用路径前缀认证（`域名/密码/目标`），认证后自动跳转至纯净链接，不污染目标参数。
- **原样透传**：对非 HTML 资源 实行 100% 二进制透传，确保数据完整性。
- **智能反馈**：异常状态下返回标准化的 JSON 错误信息，便于快速定位与调试。

## 部署步骤

1. **创建服务**：在 Cloudflare Workers 仪表盘新建 Worker。
2. **导入代码**：复制 `worker.js` 的全部内容并粘贴。
3. **配置密钥**：
   修改代码顶部的 `CONFIG` 对象：
   ```javascript
   const CONFIG = {
       SECRET: '您的密码', // 建议设置复杂密码
       // ...
   };
   ```
4. **发布上线**：点击 "Save and Deploy"。

## 使用方法

假设您的域名为 `proxy.demo.com`，密码为 `123456`。

### 1. 快捷直连（推荐）
在浏览器或书签中使用以下格式，**一次认证，长期有效**：

> **格式**：`域名/密码/目标地址`

- **示例**：`https://proxy.demo.com/123456/https://github.com`
- **效果**：自动通过鉴权并跳转至目标地址，后续访问无需携带密码。

### 2. 后台登录
- **地址**：`https://proxy.demo.com/123456`
- **效果**：直接进入代理仪表盘页面。

### 3. 普通访问
- **地址**：`https://proxy.demo.com`
- **效果**：显示极简输入框。未登录状态下输入目标地址，会自动跳转到带密码的路径进行认证。
